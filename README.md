# 커밋 규칙

```
<타입>: <변경 내용 요약>
```

커밋 타입
| 타입      | 의미                                       |
|-----------|--------------------------------------------|
| `feat`    | 새로운 기능 추가                           |
| `fix`     | 버그 수정                                  |
| `docs`    | 문서 수정 (README, 주석 등)                |
| `style`   | 코드 포맷팅, 세미콜론 누락 등 (논리 변경 없음) |
| `refactor`| 코드 리팩토링 (기능 추가/수정 없음)         |
| `test`    | 테스트 코드 추가/수정                      |
